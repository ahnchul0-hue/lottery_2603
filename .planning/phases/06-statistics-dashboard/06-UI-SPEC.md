### Accent reserved for:
- Recharts `<Bar>` fill color on the frequency bar chart (DASH-01)
- Recharts primary data series fill on ratio/range/distribution charts (DASH-04, DASH-05, DASH-06)
- NOT used for heatmap cells (those use the diverging red-blue scale)

### Heatmap Diverging Color Scale (DASH-03 only)

| Deviation | Color | RGB Formula |
|-----------|-------|-------------|
| Strong positive (over-represented) | Red | `rgb(255, 55, 55)` at +100% deviation |
| Mild positive | Light red/pink | `rgb(255, g, g)` where g = `255 * (1 - normalized)` |
| Zero (expected) | White | `rgb(255, 255, 255)` |
| Mild negative | Light blue | `rgb(r, r, 255)` where r = `255 * (1 - abs(normalized))` |
| Strong negative (under-represented) | Blue | `rgb(55, 55, 255)` at -100% deviation |

Normalization: deviation values clamped to [-1, +1] range. Values are `(actual - expected) / expected` from the backend.

Source: CONTEXT.md D-09 (red-blue diverging scheme), RESEARCH.md Pattern 3.

## Component Inventory

### New Components (This Phase)

| Component | Type | Parent | Props | Description |
|-----------|------|--------|-------|-------------|
| `StatisticsDashboard` | Container | App.tsx | `machine: string` | Orchestrates all 6 chart sections. Fetches draws from TanStack Query cache and heatmap from new API. |
| `ChartCard` | Presentational | StatisticsDashboard | `title: string, children: ReactNode` | Shared card wrapper: `bg-card rounded-xl border border-border p-4` with h3 title. Mirrors StrategySection styling. |
| `FrequencyBarChart` | Presentational | ChartCard | `data: NumberFrequency[]` | Recharts BarChart, 45 bars (numbers 1-45), blue accent fill. Height: 300px. |
| `HotColdNumbers` | Presentational | ChartCard | `hot: number[], cold: number[]` | Two rows of 10 LottoBall components each. "Hot" label above top row, "Cold" label above bottom row. |
| `HeatmapGrid` | Presentational | ChartCard | `data: HeatmapData[], selectedMachine: string` | CSS Grid: 4 rows (header + 3 machines) x 46 columns (label + 45 numbers). Inline backgroundColor per cell. Selected machine row highlighted with border-accent. |
| `RatioDistribution` | Presentational | ChartCard | `oddEven: RatioDistribution[], highLow: RatioDistribution[]` | Two side-by-side Recharts BarCharts (7 categories each: "0:6" through "6:0"). |
| `RangeDistribution` | Presentational | ChartCard | `data: ZoneDistribution[]` | Recharts BarChart, 5 zone bars (1-9, 10-19, 20-29, 30-39, 40-45). |
| `SumAcDistribution` | Presentational | ChartCard | `sumData: SumDistribution[], acData: AcDistribution[]` | Two charts side by side: total_sum histogram (20-unit bins, ~9 bars) + AC value bar chart (4 bars: ac=2,3,4,5). |

### New Hooks (This Phase)

| Hook | Purpose | Data Source |
|------|---------|-------------|
| `useStatistics(draws)` | Computes all frontend stats via useMemo: frequency, hot/cold, ratio distributions, zone distribution, sum/AC distributions | Already-cached `draws` array from `fetchMachineData` |
| `useHeatmapData()` | Fetches 3x45 heatmap deviation data from backend | `GET /api/statistics/heatmap` via TanStack Query |

### Reused Components

| Component | From | Reused For |
|-----------|------|------------|
| `LottoBall` | Phase 5 | Rendering Hot/Cold number balls in `HotColdNumbers` |
| `getLottoBallColor()` | Phase 5 | Ball color assignment inside `HotColdNumbers` |

### Container Width
- Max width: `max-w-4xl` (896px) -- matches existing App.tsx container
- Horizontal padding: `px-4` (16px each side) -- matches existing

### Chart Card Pattern
Every chart is wrapped in `ChartCard`:
```
bg-card rounded-xl border border-border p-4
  h3: text-base font-bold text-text-primary mb-4
  [chart content]
```
Gap between cards: `space-y-6` (24px)

### Chart Dimensions
| Chart | Width | Height | Notes |
|-------|-------|--------|-------|
| FrequencyBarChart | 100% (ResponsiveContainer) | 300px | 45 narrow bars |
| HotColdNumbers | 100% | auto (~120px) | Two rows of LottoBall (40px each + gap) |
| HeatmapGrid | 100% (overflow-x-auto) | auto (~100px) | 3 rows of cells + header row |
| RatioDistribution (each) | 50% (side by side) | 250px | Two charts in flex row |
| RangeDistribution | 100% (ResponsiveContainer) | 250px | 5 bars |
| SumAcDistribution (sum) | 60% | 250px | Histogram with ~9 bars |
| SumAcDistribution (AC) | 40% | 250px | Bar chart with 4 bars |

### Heatmap Grid Specifics
- Grid: `display: grid; gridTemplateColumns: 'auto repeat(45, 1fr)'`
- First column: machine label (3호기, etc.), width ~48px
- Cell columns: 45 equal-width cells (~18-19px each)
- Cell height: 24px
- Cell gap: `gap: 0` -- no gap between cells. Each cell uses `border: 1px solid var(--color-border)` to create visible grid lines between cells without introducing a non-standard spacing value.
- Overflow: `overflow-x-auto` as safety valve
- Header row: numbers 1-45, `text-[10px] text-text-secondary text-center`
- Selected machine row: `outline outline-2 outline-accent` to highlight current selection

## Interaction Contract

| Interaction | Behavior | Source |
|-------------|----------|--------|
| Machine selection changes | All dashboard charts re-compute via useMemo from new draws data. Heatmap highlights the selected machine row. No loading state needed (data already cached). | D-06, D-08 |
| Bar/point hover | Recharts default `<Tooltip>` displays value. Format: "{번호}번: {횟수}회" for frequency, "{비율}: {횟수}회" for ratios. | D-07 |
| Heatmap cell hover | Native `title` attribute shows: "{호기} - {번호}번: 편차 +12.3%" | D-07, D-10 |
| Click on any chart element | No action. Dashboard is read-only. | D-08 |
| Cross-chart interaction | None. Each chart is independent. | D-08 |
| No machine selected | Dashboard shows empty state: "호기를 선택하세요" | Design default |
| Heatmap API failure | Error text below heatmap ChartCard title. Other charts unaffected. | Design default |

## Recharts Configuration Defaults

Applied to all Recharts charts in this phase:

| Property | Value | Rationale |
|----------|-------|-----------|
| `<ResponsiveContainer>` width | "100%" | Fill card width |
| `<CartesianGrid>` strokeDasharray | "3 3" | Standard dashed grid |
| `<XAxis>` tick fontSize | 10 | 45 bars need small labels |
| `<YAxis>` width | 40 | Sufficient for count values |
| `<Tooltip>` | Default Recharts Tooltip | D-07: hover tooltips only |
| `<Bar>` fill | "#3b82f6" (accent) | Primary data series |
| `<Bar>` radius | [2, 2, 0, 0] | Slight rounding on top corners |
| Legend | None displayed | Charts are simple enough without legend |

For DASH-04 side-by-side charts, use a secondary fill color for the second chart: `#8b5cf6` (violet-500) to visually distinguish odd/even from high/low.

## Data Flow Summary

```
selectedMachine (App.tsx state)
    |
    +---> MachineSelector (existing)
    |
    +---> PredictionResults (existing, shows on predict click)
    |
    +---> StatisticsDashboard (NEW)
            |
            +---> useQuery(['machineData', machine]) --> fetchMachineData()
            |       returns draws[] (cached, staleTime=Infinity)
            |
            +---> useStatistics(draws) --> useMemo computations
            |       returns { frequencyData, hotNumbers, coldNumbers,
            |                 oddEvenDist, highLowDist, zoneDist,
            |                 sumDist, acDist }
            |
            +---> useHeatmapData() --> useQuery --> GET /api/statistics/heatmap
            |       returns HeatmapData[] (all 3 machines, cached)
            |
            +---> 6 ChartCard sections rendering computed data
```
