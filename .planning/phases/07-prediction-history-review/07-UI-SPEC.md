## Component Inventory

### New Components (This Phase)

| Component | Type | Parent | Props | Description |
|-----------|------|--------|-------|-------------|
| `HistorySection` | Container | App.tsx | `machine: string, predictionData: PredictResponse[]` | Orchestrates all history UI. Owns useHistoryStorage hook. Renders save button, performance report, history table. |
| `SavePredictionButton` | Presentational | App.tsx (near PredictionResults) | `predictions: PredictResponse[], machine: string, onSave: (roundNumber: number) => void` | Inline form: round number input (type="number", min=800, max=9999) + "예측 저장" button. Disabled when no predictions or no round number. |
| `WinningNumberInput` | Presentational | HistoryRow (accordion) | `onSubmit: (numbers: number[]) => void` | 6 input fields (w-12 h-12 each) + "비교" button. Auto-advance on complete entry. Validation: 1-45 range, no duplicates, all 6 filled. |
| `ComparisonTable` | Presentational | HistoryRow (accordion) | `comparison: ComparisonResult` | HTML table: strategy > game rows showing predicted numbers, match count, matched numbers. Per-strategy summary row with hit rate. Background tint by match count. |
| `HistoryTable` | Presentational | HistorySection | `entries: SavedPrediction[], onUpdateEntry: (id, patch) => void` | HTML table with 5 columns (D-08). Renders HistoryRow for each entry. Newest-first sort (D-10). |
| `HistoryRow` | Presentational | HistoryTable | `entry: SavedPrediction, onUpdate: (patch) => void` | Clickable table row + accordion expansion (D-09). Expanded state shows WinningNumberInput (if no comparison yet), ComparisonTable, FailureAnalysis, AiReflection. |
| `StrategyPerformance` | Presentational | HistorySection | `entries: SavedPrediction[]` | Aggregate table: per-strategy average match count, best match, total predictions (D-16). Only renders when at least 1 entry has comparison data. |
| `FailureAnalysis` | Presentational | HistoryRow (accordion) | `comparison: ComparisonResult` | Two number lists: missed numbers and overestimated numbers (D-17). Plain number display in comma-separated lists. |
| `AiReflection` | Presentational | HistoryRow (accordion) | `reflection: string, isLoading: boolean, onGenerate: () => void` | Shows "AI 분석 생성" button if no reflection. Shows loading state. Shows reflection text when available. Korean prose, rendered as whitespace-pre-wrap paragraph. |

### New Hooks (This Phase)

| Hook | Purpose | Data Source |
|------|---------|-------------|
| `useHistoryStorage()` | CRUD for localStorage prediction history. Returns `{ entries, addEntry, updateEntry }`. Lazy useState initialization for React 19 safety. | localStorage via historyStorage.ts adapter |
| `useReflection()` | TanStack Query `useMutation` wrapping POST /api/reflect. Returns mutation object for triggering AI reflection generation. | Backend POST /api/reflect endpoint |

### New Utility Modules (This Phase)

| Module | Purpose |
|--------|---------|
| `lib/historyStorage.ts` | Typed localStorage adapter: `loadHistory()`, `saveHistory()`, `STORAGE_KEY = 'lottery-prediction-history'` |
| `lib/comparison.ts` | Pure comparison functions: `comparePredictions(predictions, actualNumbers) -> ComparisonResult`. Computes match counts, strategy hit rates, missed numbers, overestimated numbers. |
| `types/history.ts` | TypeScript types: `SavedPrediction`, `ComparisonResult`, `GameComparison`, `StrategyHitRate` |

## Interaction Contract

### Save Flow

| Step | Interaction | Behavior |
|------|------------|----------|
| 1 | User runs prediction (existing) | PredictionResults renders 25 games |
| 2 | User enters round number | Numeric input field, required (D-03). Validates range 800-9999. |
| 3 | User clicks "예측 저장" | Saves to localStorage with machine, date, predictions. Button shows "저장 완료" text for 2 seconds, then resets. Button disabled if no round number or no predictions. |

### Comparison Flow

| Step | Interaction | Behavior |
|------|------------|----------|
| 1 | User clicks history row | Accordion expands in-place (D-09). If no comparison exists, shows WinningNumberInput. |
| 2 | User enters 6 winning numbers | Auto-advance on complete entry (D-05). Input validated: 1-45, no duplicates, numeric only. "비교" button disabled until all 6 valid unique numbers entered. |
| 3 | User clicks "비교" | Pure client-side comparison computed. ComparisonTable and FailureAnalysis render immediately. Comparison result saved to localStorage entry. |

### AI Reflection Flow

| Step | Interaction | Behavior |
|------|------------|----------|
| 1 | After comparison exists | "AI 분석 생성" button appears below comparison table. |
| 2 | User clicks "AI 분석 생성" | Button text changes to "AI 분석 중..." with disabled state. useMutation fires POST /api/reflect. |
| 3 | API returns reflection | Reflection text displays in AiReflection component. Saved to localStorage entry. |
| 4 | API fails | Error text: "AI 분석을 생성할 수 없습니다. API 키를 확인하세요." Button re-enables for retry. |

### Accordion Behavior

| Property | Value | Source |
|----------|-------|--------|
| Animation | None -- instant expand/collapse | Claude's discretion. Keeps implementation simple. CSS transition can be added in Phase 8 polish. |
| Multiple open | Allowed -- any number of rows can be expanded simultaneously | Design default for browsing multiple entries |
| Expand trigger | Click anywhere on the table row | D-09 |
| Collapse trigger | Click the same row again | Standard toggle behavior |
| Row hover | `hover:bg-surface` background tint | Indicates clickability |
| Expanded row indicator | Left border: `border-l-4 border-accent` on the expanded detail cell | Visual cue for active accordion |

### History Table Behavior

| Property | Value | Source |
|----------|-------|--------|
| Sort order | Newest first (descending by date) | D-10 |
| Pagination | None -- all entries shown | D-10 |
| Empty state | "저장된 예측이 없습니다" centered text with body instruction below | Design default |
| Row structure | `<tr>` for summary + conditional `<tr>` with `<td colSpan={5}>` for detail | RESEARCH.md Pattern 5 |

## Data Flow Summary

```
selectedMachine (App.tsx state)
prediction.data (App.tsx usePrediction result)
    |
    +---> PredictionResults (existing, renders 25 games)
    |
    +---> SavePredictionButton (NEW)
    |       user enters roundNumber + clicks save
    |       calls addEntry() from useHistoryStorage
    |
    +---> HistorySection (NEW)
            |
            +---> useHistoryStorage()
            |       returns { entries, addEntry, updateEntry }
            |       reads/writes localStorage
            |
            +---> StrategyPerformance
            |       aggregates across all entries with comparisons
            |
            +---> HistoryTable
                    |
                    +---> HistoryRow (for each entry)
                            |
                            +---> WinningNumberInput (if no comparison)
                            |       onSubmit -> comparePredictions() -> updateEntry()
                            |
                            +---> ComparisonTable (if comparison exists)
                            |
                            +---> FailureAnalysis (if comparison exists)
                            |
                            +---> AiReflection
                                    |
                                    +---> useReflection() useMutation
                                    |       POST /api/reflect
                                    |       sends: machine, roundNumber, comparisonData, pastReflections
                                    |       returns: reflection text
                                    |
                                    +---> updateEntry() saves reflection to localStorage
```
