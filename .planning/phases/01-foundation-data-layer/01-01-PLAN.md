    **Step 1: Create `backend/app/main.py`**
    Follow RESEARCH.md Pattern 1 exactly:
    - Import `asynccontextmanager` from `contextlib`
    - Create module-level `data_store: dict = {}`
    - Define `lifespan(app: FastAPI)` async context manager that:
      1. Creates `DataLoader(settings.DATA_PATH)`
      2. Calls `loader.load_and_validate()`
      3. Stores `data_store["loader"] = loader`
      4. Prints `f"Loaded {loader.total_records} lottery records"`
      5. On shutdown: `data_store.clear()`
    - Create `app = FastAPI(title="Lottery Predictor API", version="0.1.0", lifespan=lifespan)`
    - Add CORS middleware IMMEDIATELY after app creation (Pitfall #1):
      ```python
      app.add_middleware(
          CORSMiddleware,
          allow_origins=settings.CORS_ORIGINS,  # ["http://localhost:5173"] per D-09
          allow_methods=["GET", "POST"],
          allow_headers=["*"],
      )
      ```
    - Import and include router: `from app.api.routes import router; app.include_router(router, prefix="/api")`

    **Step 2: Create `backend/app/api/routes.py`**
    Follow RESEARCH.md Pattern 4:
    - `GET /health` (async def) -- returns `HealthResponse` with status, data_loaded, total_records from `data_store["loader"]`
    - `GET /data` (sync def, NOT async -- Pitfall #12 / future NumPy CPU-bound work) -- accepts `machine: str = Query(..., description="Machine filter: 1호기, 2호기, or 3호기")` per D-04, returns `MachineDataResponse`. Raises `HTTPException(503)` if loader not ready, `HTTPException(400)` for invalid machine.
    - Use response_model on both endpoints: `response_model=HealthResponse` and `response_model=MachineDataResponse`
