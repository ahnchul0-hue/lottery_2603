# Phase 1 — UI Design Contract

> Visual and interaction contract for the Phase 1 placeholder page. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **Scope note:** Phase 1 is backend-focused. The sole frontend deliverable is a placeholder page that proves CORS connectivity to the FastAPI backend. This contract establishes base design tokens (colors, typography, spacing, dark/light mode CSS variables) that all future phases inherit. The real UI work begins in Phase 5.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (Tailwind CSS v4 utility-first) |
| Preset | not applicable |
| Component library | none (future phases may add headless components as needed) |
| Icon library | none required in Phase 1 |
| Font | System font stack: `ui-sans-serif, system-ui, -apple-system, BlinkMacSystemFont, "Segoe UI", Roboto, sans-serif` |

**Rationale:** RESEARCH.md specifies Tailwind CSS v4 with `@tailwindcss/vite` plugin. No shadcn or component library is in the project stack. System fonts avoid external font loading for a localhost-only tool.

---

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding |
| sm | 8px | Compact element spacing, status indicator gaps |
| md | 16px | Default element spacing, card padding |
| lg | 24px | Section padding |
| xl | 32px | Layout gaps between major sections |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level vertical spacing |

Exceptions: none

**Tailwind v4 mapping:** These map directly to Tailwind's default spacing scale (`1` = 4px, `2` = 8px, `4` = 16px, `6` = 24px, `8` = 32px, `12` = 48px, `16` = 64px). No custom configuration needed.

## Color

Phase 1 establishes CSS custom properties on `:root` (light) and `.dark` (dark) for future dark/light mode toggle (UI-03, Phase 8). The placeholder page uses light mode only.

### Light Mode (default)

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#f8fafc` (slate-50) | Page background |
| Secondary (30%) | `#ffffff` (white) | Card surfaces |
| Accent (10%) | `#3b82f6` (blue-500) | Status indicator dot, success text highlight |
| Destructive | `#ef4444` (red-500) | Error state text only |
| Text primary | `#0f172a` (slate-900) | Headings, body text |
| Text secondary | `#64748b` (slate-500) | Supporting labels, "Connecting..." text |
| Border | `#e2e8f0` (slate-200) | Card border, dividers |

### Dark Mode (CSS variables only -- not toggled in Phase 1)

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#0f172a` (slate-900) | Page background |
| Secondary (30%) | `#1e293b` (slate-800) | Card surfaces |
| Accent (10%) | `#60a5fa` (blue-400) | Status indicator dot, success text highlight |
| Destructive | `#f87171` (red-400) | Error state text only |
| Text primary | `#f1f5f9` (slate-100) | Headings, body text |
| Text secondary | `#94a3b8` (slate-400) | Supporting labels |
| Border | `#334155` (slate-700) | Card border, dividers |

**Accent reserved for:** Backend connection status indicator (green dot when connected), success state text. Never for decorative elements in Phase 1.

### CSS Custom Properties Foundation

These properties MUST be declared in the main CSS file (`src/index.css`) so Phase 8 can toggle them via a `.dark` class on `<html>`:

```css
@import "tailwindcss";

@theme {
  --color-surface: #f8fafc;
  --color-card: #ffffff;
  --color-accent: #3b82f6;
  --color-destructive: #ef4444;
  --color-text-primary: #0f172a;
  --color-text-secondary: #64748b;
  --color-border: #e2e8f0;
  --color-success: #22c55e;
}
```

**Note:** Tailwind CSS v4 uses `@theme` directive for custom tokens. Dark mode values are deferred to Phase 8 implementation but documented here for planning.

## Placeholder Page Layout

Single centered card on a full-viewport background. No navigation, no sidebar, no footer.

```
+--------------------------------------------------+
|                                                    |
|                                                    |
|          +----------------------------+            |
|          |                            |            |
|          |     Lottery Predictor       |            |
|          |                            |            |
|          |     Status: Connected      |            |
|          |     Data loaded: Yes       |            |
|          |     Total records: 417     |            |
|          |                            |            |
|          +----------------------------+            |
|                                                    |
|                                                    |
+--------------------------------------------------+
```

**Layout specs:**
- Full viewport height (`min-h-screen`)
- Centered both axes (`flex items-center justify-center`)
- Card: max-width 400px, padding 32px (xl), border-radius 12px, subtle shadow (`shadow-md`)
- Card border: 1px solid `--color-border`
- Background: `--color-surface`
- Card background: `--color-card`

**States:**
1. **Loading:** Show "Connecting to backend..." in `--color-text-secondary`
2. **Success:** Show status fields with a green (`#22c55e`) dot indicator before "Connected"
3. **Error:** Show error heading in `--color-destructive` and body text with troubleshooting instruction
